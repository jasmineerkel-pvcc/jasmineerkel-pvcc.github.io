TERM DEFINITION
Machine language Programs for modern computers consist of sequences of instructions that are encoded as numeric digits. Such an encoding system is known as a machine language.
Debugging Locates and corrects errors in programs.
Assembler Once such a mnemonic system was established, programs called assemblers were developed to convert mnemonic expressions into machine language instructions.
Assembly language A mnemonic system for representing programs.
Translator Translates programs expressed in these high-level primitives into machine-language programs.
Compiler Translation programs are often called compilers.
Interpreter An alternative to translators, called interpreters, emerged as another means of implementing third-generation languages. These programs were similar to translators except that they executed the instructions as they were translated instead of recording the translated version for future use.
Cross-platform software Software that is independent of an operating system’s design as well as the machine’s hardware design and is therefore executable throughout a network.
Natural language (such as English, German, and Latin) are distinguished from formal languages (such as programming languages).
Formal languages Programming languages which evolved over time without formal grammatical analysis.
Machine independence With the development of third-generation languages, the goal of machine independence was largely achieved. Since the statements in a third-generation language did not refer to the attributes of any particular machine, they could be compiled as easily for one machine as for another.
High-level programming language High-level programming languages provide more abstractions for the programmer and make it easier for people to read and write a program.
Object-oriented programming (OOP) Programming process that object-oriented paradigm is associated with.
Scripting language A subset of the imperative programming languages is the collection of languages known as scripting languages. These languages are typically used to perform administrative tasks rather than to develop complex programs.

string
Primitive data type Data types that are included as primitives in a programming language, such as for integer and for character.
Data structure In addition to data type, variables in a program are often associated with data structure, which is the conceptual shape or arrangement of data.
Python collection data structures Lists are used to store multiple items in a single variable.
Tuples are used to store multiple items in a single variable.
Sets are used to store multiple items in a single variable.
Dictionaries are used to store data values in key:value pairs.
Function A set of instructions for performing a task that can be used as an abstract tool by other program units.
Local variable As a general rule, a variable declared within a function is a local variable, meaning that it can be referenced only within that function.
Global variable Variables whose scopes are not restricted to a particular part of a program are called global variables.
Source program The program in its original form.
Object program Translated version of source program.
